# Professional Self-Assessment
Completing the Computer Science program and developing my CS 499 ePortfolio has helped me recognize how much I have grown as a student, programmer, and future technology professional. Throughout the program, I developed skills in software development, algorithms, databases, testing, security, visualization, and professional communication. I also learned that successful computer science work involves more than writing code. It requires understanding user needs, evaluating design choices, protecting data, communicating with stakeholders, and creating solutions that are reliable and maintainable.

## Goals
My professional goal is to continue building skills in computer science and data analytics, with a special interest in sports analytics. I am interested in using data to identify patterns, improve performance, support decision-making, and communicate insights to different audiences. My coursework has helped shape this goal by giving me experience with programming, data structures, database systems, artificial intelligence, software testing, security, and visualization.

## Software Engineering
The Computer Science program helped me understand that professional software should be maintainable, reliable, usable, and adaptable. I gained experience with object-oriented programming, mobile application development, software testing, secure coding, graphics, databases, and artificial intelligence. These courses taught me to think about how different parts of a system work together rather than focusing only on individual lines of code.

This is shown in my software design and engineering enhancement, where I improved an Android Inventory App from CS 360. The original application allowed users to manage inventory items, but my enhancement made the app stronger by adding better input validation, duplicate username prevention, and a delete confirmation dialog. These changes improved the user experience and reduced the chance of errors or accidental data loss.

## Algorithms and Data Structures
My coursework also helped me build stronger problem-solving skills through algorithms and data structures. In the Vector Sorting project from CS 300, I worked with structured bid data and sorting methods. For my enhancement, I added a binary search feature, runtime measurement with C++ chrono, and defensive checks to prevent sorting and searching when no records were loaded. This helped me show an understanding of algorithmic efficiency, Big-O complexity, and design trade-offs.
For example, selection sort is easier to understand but has slower performance on larger datasets, while quicksort and binary search provide stronger effiency when data needs to be sorted and searched. This artifact shows my ability to evaluate a problem, improve the logic, and explain why the changes matter.

## Database
In CS 340, I created the Grazioso Salvare Dashboard, which connected a Python dashboard to a MongoBD database of animal shelter records. The original artifact showed that I could connect an application to a database and retrieve data for dispplay. For my enhancement, I improved the database architecture by strengthening the CRUD module, adding environment variable based configuration, improving error handling, creating reusable query methods, separating database logic from dashboard logic, and adding index support for frequently queried fields. These improvements made the project more secure, maintainable, portable, and scalable. This artifact connects to my interest in data analytics because it shows how stored data can be filtered, organized, and presented to support decision-making.

## Security
Security became an increasingly important part of my work throughout the program. I learned that security should be considered during design, development, testing, and deployment rather than added only after a project is complete. My security coursework included vulnerability assessment, dependency checking, HTTPS configuration, certificates, hashing, input validation, and secure coding practices.

These experiences taught me to look for possible weaknesses such as hard-coded credentials, invalid input, vulnerable dependencies, insecure communication, and unsafe user actions. They also helped me understand the importance of protecting user information and designing systems that respond safely when errors occur.

## Collaboration and Communication
Although many of my academic projects were completed individually, they still required me to consider the needs of users, instructors, reviewers, future developers, and other stakeholders. Code reviews, milestone feedback, testing results, and written reflections helped me evaluate my work from different perspectives. I learned to explain technical decisions in a way that could be understood by both technical and nontechnical audiences.
